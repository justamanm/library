工作目录

- 使用`os.getcwd()`获取
- pycharm中执行脚本时，工作目录即主程序所在目录（或者手动设置）
- 命令行cmd中执行时，为cmd当前所切换到的目录
- 打包后
  - 无论是主程序还是子模块中，`os.getcwd()`获取的均为cmd当前目录或调用者所在目录



绝对目录

- 使用`os.path.abspath(__file__)`获得

- 脚本执行
  - 无论在哪个目录执行脚本，主程序和子模块中的`abspath`均正常，为各自文件的绝对路径

- **打包exe后的`abspath`**（cmd中直接执行或被其他程序调用）
  - 主程序的路径变成了cmd的目录，或调用程序所在的目录
  - 子模块的路径正确，为其自身实际所在路径

- 综上
  - **`abspath`会在打包后的主程序中异常，需要使用`sys.argv[0]`**
